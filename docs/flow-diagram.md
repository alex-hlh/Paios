  aios init --defaults              → 全默认，不提问

## 4. 完整开发闭环 (standalone 模式)

```
┌─────────────────────────────────────────────────────────────┐
│                   一次完整开发任务                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户: "帮我做用户登录功能"                                    │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────────┐              │
│  │  pai:design  (需求 → 设计)                │              │
│  │                                          │              │
│  │  ① 探索项目上下文 (ai/specs/, ai/config)   │              │
│  │  ② 一问一答澄清需求 (一次一个问题)          │              │
│  │  ③ 提出 2-3 方案对比 (含利弊)             │              │
│  │  ④ 分节展示设计 (每节 200-300 字)         │              │
│  │  ⑤ 用户确认 → 写入 proposal.md + design.md│              │
│  │                                          │              │
│  │  输出: ai/changes/user-login/proposal.md   │              │
│  │        ai/changes/user-login/design.md     │              │
│  └────────────────┬─────────────────────────┘              │
│                   │ 用户确认                                 │
│                   ▼                                         │
│  ┌──────────────────────────────────────────┐              │
│  │  pai:spec  (生成规格 + 任务)              │              │
│  │                                          │              │
│  │  ① 读取 proposal + design                 │              │
│  │  ② 读取现有 ai/specs/ (识别 MODIFIED)     │              │
│  │  ③ 生成 delta spec (ADDED/MODIFIED/... )  │              │
│  │     Given/When/Then 场景格式               │              │
│  │  ④ 生成 tasks.md (每 task 2-5 分钟)        │              │
│  │  ⑤ 写入变更时间戳 (.openspec.yaml)         │              │
│  │                                          │              │
│  │  输出: ai/changes/user-login/specs/auth/   │              │
│  │        ai/changes/user-login/tasks.md      │              │
│  └────────────────┬─────────────────────────┘              │
│                   │ 用户确认                                 │
│                   ▼                                         │
│  ┌──────────────────────────────────────────────────┐      │
│  │              pai:build (TDD 循环)                  │      │
│  │                                                  │      │
│  │  ┌──────────────────────────────────────────┐    │      │
│  │  │  对每个 task:                              │    │      │
│  │  │                                          │    │      │
│  │  │  ① RED:  先写测试，确认测试失败              │    │      │
│  │  │  ② GREEN: 写最小实现，确认测试通过          │    │      │
│  │  │  ③ REFACTOR: 重构改进，测试保持通过          │    │      │
│  │  │  ④ CHECK: 勾选 tasks.md [-x]               │    │      │
│  │  │  ⑤ 触发 pai:review                        │    │      │
│  │  │                                          │    │      │
│  │  │  遵循 ai/config.yaml 的:                  │    │      │
│  │  │   缩进/引号/分号/命名/行宽/测试框架         │    │      │
│  │  │  遵循 ai/rules/ 的 80 条规则               │    │      │
│  │  └──────────────────────────────────────────┘    │      │
│  │                                                  │      │
│  │  过程中可能触发:                                   │      │
│  │  ┌──────────────────┐  ┌──────────────────┐      │      │
│  │  │ pai:debug        │  │ pai:review       │      │      │
│  │  │ ① 复现 → ② 定位  │  │ ① 完整性 (spec)   │      │      │
│  │  │ ③ 提出修复 →     │  │ ② 正确性 (逻辑)   │      │      │
│  │  │ ④ 修复并验证      │  │ ③ 合规性 (80规则) │      │      │
│  │  │ ⑤ 记录反模式      │  │ Critical → 阻断   │      │      │
│  │  └──────────────────┘  └──────────────────┘      │      │
│  └──────────────────────┬───────────────────────────┘      │
│                         │ 全部 tasks 完成                    │
│                         ▼                                   │
│  ┌──────────────────────────────────────────┐              │
│  │  pai:done  (归档收尾)                     │              │
│  │                                          │              │
│  │  ① 运行全量测试 → 全部通过                  │              │
│  │  ② 冲突检测 → 无冲突                       │              │
│  │  ③ merge delta → ai/specs/auth/spec.md    │              │
│  │  ④ 归档 → archive/2026-05-16-user-login/  │              │
│  │  ⑤ 更新状态 → current.md + tasks.md        │              │
│  │  ⑥ 生成 commit 建议 (conventional format) │              │
│  │     → git commit -m "feat(auth): add ..."  │              │
│  │  ⑦ 触发 pai:reflect                      │              │
│  └────────────────┬─────────────────────────┘              │
│                   │                                         │
│                   ▼                                         │
│  ┌──────────────────────────────────────────┐              │
│  │  pai:reflect  (自我反思)                  │              │
│  │                                          │              │
│  │  ① 回顾流程 → 有没有走偏?                  │              │
│  │  ② 意外情况 → 需要记录反模式?              │              │
│  │  ③ 技能改进 → 哪个指令需要更新?            │              │
│  │                                          │              │
│  │  写入 ai/memory/decisions.md              │              │
│  └──────────────────────────────────────────┘              │
│                                                             │
│  ✅ 完成！下一个 change 可以开始了                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────┐
│               complementary 模式 (检测到其他技能包)            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  会话启动                                                     │
│       │                                                     │
│       ▼                                                     │
│  pai:bootstrap 检测到:                                        │
│    - 技能重叠: brainstorming (Superpowers)                   │
│    - 项目痕迹: openspec/                                    │
│       │                                                     │
│       ▼                                                     │
│  用户选择: B) 互补模式                                         │
│       │                                                     │
│       ├── AIOS 注入 ──────────────────────┐                  │
│       │   ✅ L1 红线 (8条)                 │                  │
│       │   ✅ 项目规则 (80条)                │  背景约束         │
│       │   ✅ Red Flags 表                  │  全程生效         │
│       │   ✅ 代码规范 (config.yaml)         │                  │
│       │   ✅ 项目状态 (ai/state/)          │                  │
│       └──────────────────────────────────┘                  │
│                                                             │
│       ├── 其他技能包驱动工作流 ──────────────┐                  │
│       │   Superpowers 技能链:                │                  │
│       │   brainstorming → plans → TDD       │                  │
│       │   → review → finish                 │                  │
│       │                                     │                  │
│       │   OpenSpec 管理:                     │                  │
│       │   opsx:propose → apply → archive    │                  │
│       └─────────────────────────────────────┘                  │
│                                                             │
│       └── AIOS 不注入 ──────────────────────┐                  │
│           ❌ 不注入技能链触发规则             │                  │
│           ❌ 不覆盖其他包的 Red Flags            │                  │
│           ❌ 不干扰其他包的工作流              │                  │
│           └──────────────────────────────────┘                  │
│                                                             │
│  效果: AIOS 作为"背景规则引擎"运行                              │
│    - 你写代码时 → 80条规则约束代码质量                           │
│    - 你调试时 → L1 红线阻止危险操作                              │
│    - 你提交时 → git-rules 检查 commit 格式                     │
│    - 工作流 → 由 Superpowers/OpenSpec 驱动                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
