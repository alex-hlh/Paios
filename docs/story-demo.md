## 第二步：数据流

```
AI:  我扫描了原型图，识别出可操作元素：
      登录页：邮箱输入框、密码输入框、登录按钮、忘记密码链接
      注册页：昵称/邮箱/密码/确认密码输入框、注册按钮

    写入 data-flows.md：
```

**`docs/data-flows.md`**：

```
## 登录页数据流

[邮箱输入框] → 收集 email
[密码输入框] → 收集 password
[登录按钮] ──点击──→ login(email, password)
                       │
                       ├─ 成功 → { token, user }
                       │        → 存储 token，跳转首页
                       │
                       └─ 失败 → { error }
                                → 显示"邮箱或密码错误"

## 注册页数据流

[注册按钮] ──点击──→ 前端校验(必填,邮箱格式,密码一致)
                       │
                       ├─ 不通过 → 逐字段提示
                       └─ 通过 → register({昵称,邮箱,密码})
                                   │
                                   ├─ 成功 → 自动登录跳转
                                   └─ 失败 → 显示错误
```

确认: 数据流覆盖了所有交互？[Y/n] __你: Y__

## 第五步：功能点汇总

```
AI:  综合前面 4 步，生成 docs/feature-points.md：

## 前端
├─ 登录页 (邮箱/密码/提交/忘记密码/三方登录)
├─ 注册页 (昵称/邮箱/密码/确认密码/提交)
├─ 表单验证 (空值/邮箱格式/密码一致)
├─ 调用 login API / register API
└─ token 存储 + 页面跳转

## 后端
├─ POST /api/auth/login  (公开)
├─ POST /api/auth/register (公开)
├─ bcrypt 密码哈希验证
├─ JWT 生成 (HS256, 24h)
├─ 登录失败限制 (5 次锁定 10 分钟)
└─ User 表 (id, nickname, email, password_hash)
