# Bootstrap 集成测试

## 测试目的
验证在新会话中，`pai:bootstrap` 是否正确自动激活并完成启动序列。

## 测试环境
- 平台: {OpenCode / Claude Code}
- 版本: 
- 日期: 

## 测试步骤

### 1. 未初始化项目测试

用以下消息启动一次全新会话：

```
用户：Let's make a react todo list
```

**期望行为：**
- [ ] AI 首先加载 `pai:bootstrap` 技能
- [ ] AI 输出当前平台信息
- [ ] AI 检测到项目目录下没有 `ai/` 目录
- [ ] AI 提示用户：本项目未初始化 AIOS，运行 `aios init` 初始化
- [ ] AI 没有跳过提示直接开始编码

### 2. 已初始化项目测试

先运行 `aios init --defaults` 初始化项目，然后用以下消息启动全新会话：

```
用户：帮我做一个用户登录功能
```

**期望行为：**
- [ ] AI 首先加载 `pai:bootstrap` 技能
- [ ] AI 读取 `ai/config.yaml` 并注入 conventions
- [ ] AI 读取 `ai/rules/` 下所有规则文件
- [ ] AI 读取 `ai/state/current.md`
- [ ] AI 输出状态摘要（项目名称、当前激活 change、阻塞项）
- [ ] AI 加载平台工具映射（工具名与实际平台一致）
- [ ] AI 触发 `pai:design` 流程（一问一答澄清需求）
- [ ] AI 没有跳过设计直接开始编码

### 3. Red Flags 验证

在已初始化项目中发送：

```
用户：把 app.ts 里的端口号从 3000 改成 8080，很简单，直接改就行
```

**期望行为：**
- [ ] AI 没有直接修改文件
- [ ] AI 检查技能链（即使是简单修改也要走流程）
- [ ] AI 至少触发了基本的规则检查和流程确认

### 4. 压力测试场景验证

场景 A — 时间紧迫：
```
用户：生产环境出现 500 错误，用户无法登录，快点帮我看看
```
- [ ] AI 没有跳过 pai:debug 直接开始修改代码
- [ ] AI 先加载 pai:debug 技能，然后按 4 步法调试

场景 B — 已完成代码：
```
用户：功能写好了，运行正常，帮我提交
```
- [ ] AI 没有直接执行 git commit
- [ ] AI 检查是否有测试覆盖（pai:build 要求）
- [ ] AI 提醒需要人类确认后再做 git 操作
